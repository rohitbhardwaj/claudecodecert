# One-Page Handout — Design the Approval Checkpoint

## Key Lesson

```text
Approval is not a meeting. It is evidence that the right risks were understood before code changed.
```

## Approval Must Cover

```text
business behavior
money/refund logic
payment gateway behavior
API contract
data/repository impact
logging/security
tests
rollback
PR slicing
owners
```

## Best Approval Artifact

```text
A plan listing behavior, risks, tests, owners, and rollback.
```

## Weak Approval Artifact

```text
Looks good.
```
