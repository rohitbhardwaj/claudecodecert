# Scoring Rubric

## 5 — Excellent

Participant identifies skipped planning, missing context, missing owners, unsafe tests, behavior risk, missing rollback, and writes a strong planning checkpoint.

## 4 — Strong

Participant finds most workflow failures and blocks the PR.

## 3 — Acceptable

Participant finds implementation issues but misses some planning/approval details.

## 2 — Weak

Participant focuses mostly on code correctness and not workflow.

## 1 — Poor

Participant approves because happy-path tests passed.
