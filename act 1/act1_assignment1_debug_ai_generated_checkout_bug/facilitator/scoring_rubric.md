# Scoring Rubric

## 5 — Excellent

Participant finds all major behavior changes, blocks the PR, critiques the prompt, and proposes an analysis-first safer workflow.

## 4 — Strong

Participant finds most behavior changes and blocks the PR.

## 3 — Acceptable

Participant finds some bugs but misses one or two governance lessons.

## 2 — Weak

Participant focuses on code style and misses production behavior risk.

## 1 — Poor

Participant approves the PR or treats test removal as acceptable.
