# Completed Prompt Worksheet — Strong Example

| Prompt requirement | What should the prompt say? |
|---|---|
| Do not edit yet | Do not modify files yet. |
| Inspect relevant files | Inspect CLAUDE.md, CheckoutService, CouponPolicy, EnterpriseCustomerPolicy, PaymentResultMapper, tests, API contract, and incident notes. |
| Preserve behavior | Preserve coupon expiration, enterprise eligibility, payment error codes, and reason-code logging. |
| Identify tests | Identify existing failure-path tests and tests to add before code changes. |
| Identify risk areas | Call out risks where simplification could change behavior. |
| Identify approval owners | List checkout, payment, API, QA, and architecture owners if approval is needed. |
| Define rollback | Include rollback considerations for checkout behavior change. |
| Wait for approval | Return plan and wait for approval before editing files. |
