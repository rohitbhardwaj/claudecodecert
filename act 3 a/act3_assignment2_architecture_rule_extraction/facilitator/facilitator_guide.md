# Facilitator Guide — Architecture Rule Extraction

## Learning Objective

Participants learn how to turn messy repo patterns and old review comments into clear repo-readable rules.

## Setup Script

Say:

```text
Claude Code cannot follow architecture rules that only exist in people’s heads.
Your job is to extract the rules from messy code and old PR comments and put them where Claude and developers can use them.
```

## 8-Minute Live Version

| Segment | Time |
|---|---:|
| Explain scenario | 1 min |
| Review messy patterns | 2 min |
| Fill worksheet | 3 min |
| Review strong answer | 1 min |
| Certification question | 1 min |

## 20-Minute Workshop Version

| Segment | Time |
|---|---:|
| Inspect source files | 5 min |
| Inspect old PR comments | 4 min |
| Extract rules | 5 min |
| Draft CLAUDE.md section | 4 min |
| Debrief | 2 min |

## Facilitation Prompts

Ask:

```text
What review comment keeps repeating?
What rule would prevent it?
Where should the rule live?
How would Claude understand the rule?
How would reviewers verify it?
```

## Strong Closing Line

Good Claude Code governance converts tribal architecture knowledge into repo-readable rules.
