# Scoring Rubric

## 5 — Excellent

Participant identifies weak repo guidance, wrong layer, policy bypass, refund duplication, flow inconsistency, reason-code drift, approval gap, test gap, and writes strong `CLAUDE.md` rules.

## 4 — Strong

Participant finds most issues and blocks the PR.

## 3 — Acceptable

Participant identifies wrong layer but misses ownership or test governance.

## 2 — Weak

Participant focuses only on code correctness and misses repo guidance.

## 1 — Poor

Participant approves the happy-path controller implementation.
