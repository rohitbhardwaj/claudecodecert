# Scoring Rubric

## 5 — Excellent

Participant identifies the wrong-layer fix, root cause in CouponPolicy, all affected flows, architecture drift, missing tests, and blocks the PR.

## 4 — Strong

Participant finds the root cause and most missing flow coverage.

## 3 — Acceptable

Participant finds the controller/domain issue but misses batch or test coverage.

## 2 — Weak

Participant sees the bug but does not identify the shared source of truth.

## 1 — Poor

Participant approves the web-only fix.
