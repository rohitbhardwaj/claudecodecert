# Scoring Rubric

## 5 — Excellent

Participant includes all entry points, shared policies, pricing, API, tests, logging, rollback, ownership, and repo rules.

## 4 — Strong

Participant includes most context but misses one supporting artifact.

## 3 — Acceptable

Participant includes core source files but misses governance or rollout files.

## 2 — Weak

Participant focuses mostly on controllers and tests.

## 1 — Poor

Participant includes only the web checkout file.
